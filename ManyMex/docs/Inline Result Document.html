Inline Result Document

Represents a link to a file. By default, this file will be sent by the user with an optional caption. Alternatively, you can use input_message_content to send a message with the specified content instead of the file. Currently, only .PDF and .ZIP files can be sent using this method.

Field Type Optional Description
type String No Type of the result, must be document
id String No Unique identifier for this result, 1-64 bytes
title String No Title for the result
caption String Yes Caption of the document to be sent, 0-1024 characters
parse_mode String Yes Send Markdown or HTML, if you want Telegram apps to show bold, italic, fixed-width text or inline URLs in the media caption.
document_url or document_file_id String No A valid URL for the file or a valid file identifier for the file
mime_type String No Mime type of the content of the file, either "application/pdf" or "application/zip"
description String Yes Short description of the result
reply_markup Inline Keyboard Yes Inline keyboard attached to the message
input_message_content Message Content Yes Content of the message to be sent instead of the file
thumb_url String Yes URL of the thumbnail (jpeg only) for the file
thumb_width Integer Yes Thumbnail width
thumb_height Integer Yes Thumbnail height
